## 🧠 模型选择思路

&#x20;

这个数据集是典型的金融时间序列，特点是：有趋势、有波动性、非线性、可能有周期性。综合考虑项目要求（多模型比较）和实现难度，建议用以下组合：

| 模型                | 理由                      | 难度  |
| :---------------- | :---------------------- | :-- |
| Linear Regression | 基础对照组，解释性强，体现对比价值       | ⭐   |
| ARIMA             | 时间序列经典模型，必选，学术认可度高      | ⭐⭐  |
| Prophet           | Meta 出品，自动处理趋势+节假日，适合股票 | ⭐⭐  |
| LSTM              | 深度学习，捕捉复杂非线性，是亮点        | ⭐⭐⭐ |

四个模型覆盖了"传统统计 → 机器学习 → 深度学习"的完整梯度，比较维度丰富，报告论述空间大。

## 📋 项目大纲

### 第一步：数据准备

* 选定 1-3 只有代表性的股票（建议：AAPL / TSLA / SPY ETF，覆盖科技股+指数）

* 加载并合并 CSV，统一时间索引

* 检查数据范围、时间连续性

### 第二步：EDA（探索性分析）

* 绘制收盘价历史走势图

* 计算滚动均线（MA7、MA30）

* 分析成交量与价格的相关性

* 检验平稳性（ADF test）—— 为 ARIMA 做准备

### 第三步：数据预处理

* 缺失值处理（交易日空缺用前值填充）

* 特征构造（供 Linear Regression 使用）：

  * Lag features（前1/5/20日收盘价）

  * 移动平均、收益率

* 划分训练集（80%）/ 测试集（20%），严格按时间顺序切分

* 归一化（MinMaxScaler，供 LSTM 使用）

### 第四步：模型实现

① Linear Regression

* 输入：Lag features + 技术指标

* 输出：次日收盘价

② ARIMA

* ACF/PACF 图确定 p、d、q 参数

* 或用 auto_arima 自动选参

③ Prophet

* 输入格式：ds（日期）、y（收盘价）

* 加入 add_country_holidays(country_name='US')

④ LSTM

* 滑动窗口构造序列（window=60天）

* 网络结构：2层 LSTM + Dropout + Dense

* 用 Keras 实现

### 第五步：模型评估与比较

* 指标：RMSE、MAE、MAPE（三个都算，报告更丰富）

* 制作统一对比表格

* 可视化：4个模型预测值 vs 实际值叠加图

### 第六步：预测与结论

* 用最优模型预测未来 30 天走势

* 分析各模型失败原因（金融市场不可预测性）

* 给出建议（如：LSTM 精度最高但计算成本最大，实际应用中 Prophet 性价比更高）

* PPT 精简为 10-15 页，4分钟英文讲解
